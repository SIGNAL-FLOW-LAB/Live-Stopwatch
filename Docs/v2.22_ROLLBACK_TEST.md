# v2.22 Rollback Test

## 方針

表示品質が良かったv2.0へ時計表示を戻し、
変更範囲を限定した検証版です。

## 時計表示

- SwiftUIの大型Text表示へ復帰
- v2.0と同じウィンドウ連動の拡大縮小
- AppKit NSTextField方式を撤回
- 時計更新は10回/秒
- 表示される0.1秒が変わったときだけ更新

## 維持した改善

- MIDIControllerの多重起動防止
- UDP Listener、Timer、CoreMIDIの重複起動防止
- アプリ内Remote Scriptインストーラー
- アプリアイコン
- タイトルバーのアプリ名

## SELECTED初期表示

起動時にLiveが保持していたTrack 1のハイライト位置は無視します。
起動後にユーザーが実際にClip Slotを変更するまでは、
SELECTEDにはMain列のScene名を表示します。

## 確認項目

1. 時計の大きさがv2.0と同等か
2. ウィンドウ拡大・縮小に自然に追従するか
3. 初回Main列選択でScene名になるか
4. 通常トラックの別クリップを選ぶとクリップ名になるか
5. Intel MacでCPU負荷が時間経過で増え続けないか
